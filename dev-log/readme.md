# .dev-log/ — Журнал разработки для агентов

> Эта папка — **память проекта**. Каждый агент, вносящий изменения, обязан оставить запись.
> Агент-аудитор/тестер читает этот журнал, чтобы понять что изменилось, где искать баги и как связаны компоненты.

---

## Правила

### Когда писать запись

**После каждого коммита** (или логически завершённой итерации) агент создаёт файл в этой папке.

### Формат имени файла

```
YYYY-MM-DD_HH-MM_<scope>_<slug>.md
```

Примеры:
```
2026-03-11_14-30_api_webhook-registration.md
2026-03-11_16-00_web_overview-page.md
2026-03-12_09-15_prisma_add-custom-labels.md
2026-03-12_11-00_api_attribution-algorithm.md
```

### Структура записи

Каждый файл следует шаблону из `_TEMPLATE.md`. Обязательные секции:

1. **Что сделано** — краткое описание (2–5 предложений)
2. **Затронутые файлы** — полные пути к изменённым/созданным файлам
3. **Зависимости** — какие пакеты добавлены, какие сервисы зависят от изменения
4. **Связи** — что от чего зависит, что может сломаться
5. **Тесты** — какие тесты написаны/обновлены, все ли прошли
6. **Известные ограничения** — что не доделано, workarounds, TODO

### Кто читает

- **Агент-разработчик** — перед началом работы, чтобы понять текущее состояние
- **Агент-аудитор/тестер** — чтобы найти что тестировать и где искать баги
- **Человек** — чтобы быстро понять прогресс без чтения git log

---

## Структура папки

```
.dev-log/
├── README.md              ← этот файл
├── _TEMPLATE.md           ← шаблон для новых записей
├── _INDEX.md              ← автоматически обновляемый индекс всех записей
└── 2026-03-11_14-30_api_webhook-registration.md
└── ...
```

---

## Для агента-аудитора / тестера

Если ты агент, проверяющий код:

1. **Начни с `_INDEX.md`** — там сводка всех изменений в хронологическом порядке
2. **Читай записи от новых к старым** — последние изменения = самые вероятные источники багов
3. **Обращай внимание на секцию "Связи"** — там указано что может сломаться
4. **Проверяй секцию "Известные ограничения"** — там честно написано что не доделано
5. **Создай свою запись** после аудита с найденными проблемами
